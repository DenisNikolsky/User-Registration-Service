# 📦 User Registration Service with Async API, Caching & Message Queues

- Асинхронное REST API с валидацией данных
- Хеширование паролей (bcrypt)
- Постоянное хранение в SQLite (aiosqlite)
- Кэширование данных в Redis (cache‑aside)
- Асинхронная обработка фоновых задач через RabbitMQ (отправка приветственных писем)
- Полная контейнеризация (Docker + docker‑compose)

Проект написан на **Python 3.11+** и предназначен для демонстрации навыков, необходимых бэкенд-разработчику в высоконагруженных проектах.



## Основные возможности

- **Регистрация пользователей**  
  - POST `/users/` – принимает `name`, `age`, `password`  
  - Пароль хешируется перед сохранением  
  - Проверка уникальности имени

- **Получение данных пользователя**  
  - GET `/users/{name}` – возвращает `name` и `age` (пароль не передаётся)  
  - **Кэширование в Redis** (TTL 60 сек) – повторные запросы выполняются мгновенно

- **Список всех пользователей**  
  - GET `/users/all_users` – возвращает массив `{name, age}`

- **Очистка кэша**  
  - GET `/users/{name}/cache` – удаляет запись пользователя из Redis (для отладки)

- **Фоновая обработка**  
  - При регистрации в очередь RabbitMQ отправляется событие `user_registered`  
  - Отдельный воркер слушает очередь и эмулирует отправку приветственного письма



## Технологический стек

| Компонент           | Технология                        |
|---------------------|-----------------------------------|
| Web-фреймворк       | FastAPI (асинхронный)             |
| Сервер              | Uvicorn                           |
| База данных         | SQLite (aiosqlite)                |
| Хеширование         | bcrypt                            |
| Кэширование         | Redis (redis-py, asyncio)         |
| Брокер сообщений    | RabbitMQ (aio-pika)               |
| Контейнеризация     | Docker, docker-compose            |
| Язык                | Python 3.11+                      |



## Требования для локального запуска

- Python 3.11 или выше
- Установленный [Docker](https://www.docker.com/) и Docker Compose
- Git (для клонирования)

## Установка и запуск

### Клонируйте репозиторий
```bash
git clone https://github.com/your-username/your-repo.git
cd your-repo
docker-compose up --build
```

### После успешной сборки:

API будет доступно на http://localhost:8000

Swagger-документация – http://localhost:8000/docs

RabbitMQ Management – http://localhost:15672 (логин guest / пароль guest)

База данных users.db сохраняется в текущей директории, поэтому данные не теряются при перезапуске контейнеров.
